Declarations

Names

In Cabin, we call variables names. This is because names are often unchanging, and therefore by definition, not variable. Names can be declared with the let keyword, for example: 

				let message = "Hello world!";

Names always reference a value. Two names can reference the same value!

				let message = "Hello world!";
				let message2 = message;

In this example, changing the value of either name will also change the value of another name. However, Cabin won't actually let you write this implicitly. You can't directly assign one name to another, you need to specify if you want to reference it or copy it. 

				let message = "Hello world!";
				let message2 = message; // This isn't valid!
				let message3 = message.copy(); // This IS valid!
				let message4 = message.reference(); // This is also valid!

The reference function exists on all values, and it actually does nothing! This can seem somewhat silly, but it conveys intention and prevents you from misunderstanding assignments. Cabin will not let you raw assign a name to a name, because it could cause confusion as to what's happening.

Objects

Objects are groups of values. In Cabin, you can create an object with the new keyword:

			let john = new Object {
				name = "John",
				age = 30,
				address = new Object {
					street = "Main St",
					house = 123
				}
			};

You can access names of this object with . syntax:

			let john = new Object {
				name = "John",
				age = 30,
				address = new Object {
					street = "Main St",
					house = 123
				}
			};

			terminal.print(john.name); // John
			terminal.print(john.address.street); // Main St
