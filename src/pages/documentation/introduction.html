Introduction

Welcome to Cabin! Cabin is a general-purpose programming language centered around simplicity and developer experience.

Purpose & Philosophy

It's not a stupid question to ask why Cabin exists. We have so many programming languages now covering every use case imaginable — why do we need another competing standard?

Cabin is specifically designed to fulfill the three S's: simple, safe, speedy . Cabin was created under the beliefs that very few, if any languages, succeed at excelling in all three of these values: Cabin is designed to be safer than C, simpler than Rust, and faster than Go.

These are all languages also built around these three ideas.

In short, Cabin's philosophy consists of the following ideals:

  • Stupidly obvious solutions are better than clever tricky ones.
    • Cabi
  • The most common use case should be the default.
  • What can be done with existing syntax readably should be done with existing syntax.
    • n
  • Code should do what you expect it to from a cursory glance.

Here's a few concrete ways Cabin implements these abstract philosophies:

  • Cabin has extremely little syntax.
  • Strings are multiline and formatted by default.
  • All declarations share the same syntax.
  • Cabin will never perform an implicit copy.

Installation

You can install Cabin using the default method for your operating system as shown on the home page . For more detailed instructions, see below.

Universal

Cabin is written in Rust, and minimally supported by all operating systems that support Cargo. You can install Cabin on any operating system (that supports Cargo) by using Cargo:

$ cargo install cabin-language

Testing

You can test if Cabin is installed by running cabin in the terminal. It should print a help and usage message about the Cabin compiler. If you get some sort of error or unrecognized command, continue onto the troubleshooting section.

Your First Program

Now that you've got Cabin installed, you're ready to get coding! Create a file ending in .cabin and add the following contents: 

			run terminal.print("Hello world!");

You're done! You've created your first Cabin program. You can run it and see it's output by running cabin run <filename>.cabin : 

			$ cabin run script.cabin
			Hello world!
			$

Compiler Commands

The Cabin compiler is designed to be simple but powerful, just like Cabin itself. There are just a few core commands availble:

The New Command

The first command to know is cabin new It takes no positional arguments, and creates a new Cabin project . A Cabin project comes with a src/ directory with a main.cabin file in it, containing the same hello world program you saw above. It also comes with a cabin.toml file in the root of the project, which contains information about the project, such as the name, dependencies, etc.

cabin new isn't necessary to use Cabin. You can just as easily create a single Cabin file and run it and build it. cabin new is designed for larger projects and public libraries, which need a more structured approach to their code.

The Run Command

Hey, we've seen this one before! The cabin run command will run a single Cabin file or an entire project. If you want to run a single Cabin file, you must specify the file to run, i.e., cabin run hello_world.cabin . However, in a project, if you're cd 'd into the project's directory, you can run the whole project with just cabin run . Your program will be run with src/main.cabin as it's entry point, or a different entry point specified in cabin.toml or at the command-line.

The Build Command

Another useful command is cabin build . It acts similarly to cabin run , except it outputs a permanent executable file, and doesn't run it automatically. This is typically used to deploy your product once it's complete. cabin build follows the same entry point conventions as cabin run . The output will be placed in builds/ .

The Format Command

cabin format will format your entire project, or a single file if given. Cabin's formatter is opinionated and does not take any configuration options.

The Add Command

The cabin add command will add a dependency library to your project.

The Check Command

The cabin check command will lint a project or file. The default output is human-readable, which outputs pretty code snippets and colors that point right at the errors. Other output formats, such as TOML, JSON, and YAML, are supported through flags.

The Convert Command

Cabin can be transpiled to C. The cabin convert command does just that.

Project Configuration

Cabin projects come with a cabin.toml file in the root of the project. This file provides a bunch of information to the compiler, such as project information and compilation flags. Below is an example cabin.toml demonstrating all possible values for the configuration file: 

			[information]
			name = "example_project"

			[options]
			quiet = true

We can go through the configuration options one-by-one:

  • [information]
    • name [string] - The name of the Cabin project. If you plan to upload this as a public library, this is the name that will be used to reference the project as a dependency.
  • [options] - options and flags passed to the compiler
    • quiet [bool] - Uses "quiet mode" for the compiler. This will prevent the compiler from printing progress information during compilation.
